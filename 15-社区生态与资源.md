## 1. Hermes 社区全景

### 1.1 社区规模与活跃度

如果说 Hermes Agent 是一棵正在茁壮成长的树，那么社区就是滋养它的土壤。没有活跃的社区，再好的技术也只是一潭死水。

截止到 2026 年中，Hermes Agent 社区已经发展成了一个相当有活力的生态：

| 指标 | 数据（约） |
|------|-----------|
| GitHub Stars | 15,000+ |
| Discord 成员 | 8,000+ |
| 中文微信群/飞书群 | 20+ 个群，覆盖 5000+ 人 |
| 技能商店 Skills | 300+ 个 |
| 生态项目 | 100+ 个 |
| 活跃贡献者 | 200+ 人 |

这些数据放在整个 AI Agent 领域来看，虽然比不上 AutoGPT、LangChain 这些"老牌明星"，但考虑到 Hermes Agent 是后起之秀，这个增长速度已经相当可观了。

> **Tips**：判断一个开源项目是否值得投入时间，最重要的指标不是 Star 数，而是**活跃度**。建议你去 GitHub 看看最近一个月的 Commit 频率、Issue 的响应速度、PR 的合并速度——这些才是真正的健康指标。

### 1.2 核心贡献者与维护团队

Hermes Agent 最初由 **Nous Research** 发起和维护。Nous Research 是一家专注于开源 AI 研究的机构，在 AI 社区中颇有声望。

**核心维护团队包括：**
- **创始人/核心架构师**：负责整体技术路线和产品方向
- **核心开发者**：负责主要模块的开发和维护
- **社区经理**：负责协调社区活动、处理社区事务
- **文档维护者**：负责官方文档的撰写和更新

除了官方团队，社区中还涌现了一大批**明星贡献者**：
- 开发高质量 Skill 的"技能大神"
- 写教程、做视频的"布道者"
- 回答新手问题的"热心老哥"
- 提交 Bug 修复和功能改进的"代码侠客"

> **注意**：开源社区是" Meritocracy "（ meritocracy 精英制）——你的影响力不取决于你的头衔，而取决于你的贡献。一个刚入门的新手，如果提交了一个高质量的 PR，同样会受到社区的尊重。

### 1.3 社区文化与价值观

每个社区都有自己的"气质"。Hermes 社区的文化可以概括为几个关键词：

**"Stay Lazy, Stay Early"**
这是 Hermes 社区最流行的 Slogan。什么意思呢？
- **Stay Lazy**：用 AI 自动化一切能自动化的工作，把精力留给真正重要的事
- **Stay Early**：保持对新事物的敏感度，做早期采用者（Early Adopter）

**开放与分享**
社区鼓励成员分享自己的 Skill、工作流和经验。你会发现，很多大神会毫无保留地开源自己的项目——因为在 Hermes 社区，"分享"本身就是一种声望积累。

**务实主义**
Hermes 社区不太喜欢"空谈概念"，更喜欢"动手实干"。一个能用的 Demo 比一百页 PPT 更有说服力。

**包容新手**
虽然社区里有很多技术大牛，但对待新手的态度非常友好。你可以在 Discord 或微信群里问任何"傻问题"，通常都会有人耐心解答。

> **常见问题**：我英语水平不好，能参与国际社区吗？
> 
> 当然可以！Hermes 社区对非英语母语者非常包容。你可以在 Discord 的中文频道交流，也可以先用翻译工具辅助阅读。事实上，社区里有很多来自中国的活跃贡献者。

## 2. 官方资源大全

### 2.1 GitHub 仓库

**地址：https://github.com/NousResearch/hermes-agent**

GitHub 是 Hermes 的大本营，所有源代码、Release 版本、Issue 讨论都在这里。

**仓库结构速览：**
```
hermes-agent/
├── README.md              # 项目介绍和快速开始
├── docs/                  # 官方文档
├── src/                   # 核心源代码
├── skills/                # 内置技能示例
├── tests/                 # 测试用例
├── examples/              # 使用示例
├── CONTRIBUTING.md        # 贡献指南
├── LICENSE                # 开源协议
└── pyproject.toml         # 项目配置
```

**新手必看的几个页面：**
1. **README.md**：项目的"门面"，包含安装指南和快速开始
2. **Releases 页面**：下载稳定版本，查看更新日志
3. **Issues 页面**：搜索你遇到的问题，看看有没有人已经提过
4. **Discussions 页面**：技术讨论、使用心得交流

> **Tips**：建议每个新手都先给 GitHub 仓库点个 Star，这样有更新时你会收到通知。同时，Watch 仓库可以及时收到 Issue 和 PR 的动态。

### 2.2 官方英文文档

**地址：https://hermes-agent.nousresearch.com/docs/**

官方文档是最权威的技术参考，涵盖：
- 安装与配置指南
- 核心概念解释
- Skill 开发教程
- API 参考手册
- 高级主题（自定义工具、记忆系统、安全等）

**文档使用技巧：**
- 左侧有目录导航，按主题查找
- 顶部有搜索框，可以直接搜索关键词
- 很多页面底部有"Edit this page"链接，可以直接提交文档改进

> **注意**：官方文档是英文的。如果你的英文阅读有困难，可以配合浏览器翻译插件使用。同时，中文社区也有翻译版文档（见下文）。

### 2.3 中文社区文档

**文档站 1：https://hermes-doc.aigc.green/**

这是由中文社区志愿者维护的文档翻译站，内容主要来自官方文档的中文翻译，同时加入了一些中文用户特有的使用技巧。

**特点：**
- 与官方文档结构对应，方便对照阅读
- 加入了很多中文用户的实战案例
- 更新频率较高，基本跟随官方版本

**文档站 2：https://hermesagent.org.cn**

这是另一个中文文档站，由不同的社区团队维护。

**特点：**
- 界面更友好，适合纯新手
- 有很多图文并茂的教程
- 包含一些视频教程的链接

> **Tips**：建议两个中文文档站都收藏，因为它们的侧重点不同。遇到问题时，如果一个站没找到答案，可以去另一个站试试。

### 2.4 官方技能商店

**地址：https://hermes-agent.nousresearch.com/docs/skills**

Skills Hub 是 Hermes 官方的 Skill 分发平台。你可以在这里：
- 发现和下载社区贡献的 Skill
- 上传自己开发的 Skill
- 查看 Skill 的使用统计和评分

**如何使用 Skills Hub：**
```bash
# 从 Skills Hub 安装 Skill
hermes skill install author/skill-name

# 查看已安装的 Skill
hermes skill list

# 更新 Skill
hermes skill update skill-name
```

### 2.5 Hermes Atlas（生态地图）

**地址：https://hermes-ecosystem.vercel.app**

Hermes Atlas 是一个可视化生态地图，展示了整个 Hermes 生态系统的项目分布。

**Atlas 上可以看到：**
- 各类生态项目的分类和介绍
- 项目之间的关联关系
- 热门项目排行榜
- 最新加入的项目

> **Tips**：Atlas 是发现新项目的好地方。建议每个月去逛一次，看看有没有什么新工具可以试用。

### 2.6 Awesome 资源列表

**地址：https://github.com/0xNyk/awesome-hermes-agent**

这是一个社区维护的 Awesome List，收集了大量与 Hermes 相关的优质资源。

**列表包含：**
- 官方资源
- 社区项目
- 教程和文章
- 工具和插件
- 视频和播客

> **Tips**：Awesome List 是开源社区的传统。几乎每个热门项目都有一个对应的 Awesome List，是快速了解生态全貌的好工具。

### 2.7 橙皮书（花叔）

**地址：https://www.huasheng.ai/orange-books/hermes-agent/**

橙皮书是由社区知名贡献者"花叔"编写的一系列 Hermes Agent 教程和指南。

**特点：**
- 中文原创，语言通俗易懂
- 有很多实战案例和踩坑经验
- 更新频繁，紧跟版本变化

> **Tips**：花叔的橙皮书是很多中文用户的"入门圣经"。如果你刚接触 Hermes，建议先从橙皮书看起。

### 2.8 Release Notes 与 Changelog

**位置：** GitHub 仓库的 Releases 页面

每次新版本发布时，官方都会发布详细的 Release Notes，包含：
- 新增功能
- Bug 修复
- 性能改进
- 破坏性变更（Breaking Changes）
- 升级指南

> **注意**：升级 Hermes 版本前，务必阅读 Release Notes 中的 Breaking Changes 部分。有些升级可能需要你修改配置文件或 Skill 代码。

## 3. 中文社区资源

### 3.1 中文文档站

除了前面提到的两个中文文档站（hermes-doc.aigc.green 和 hermesagent.org.cn），还有一些其他的中文资源：

**语雀文档**
一些团队使用语雀维护 Hermes 相关的文档和教程，搜索"Hermes Agent"可以找到不少。

### 3.2 微信群与飞书群

中文社区最活跃的交流场所在微信群和飞书群。

**如何加入：**
1. 关注 Hermes 相关的公众号或小红书账号
2. 通常会在文章末尾放群二维码或邀请方式
3. 也可以去 GitHub Discussions 的中文板块询问

**群里的日常：**
- 新手提问（安装、配置、报错）
- 使用技巧分享
- 新功能讨论
- 项目展示和招聘

> **Tips**：进群后建议先潜水观察几天，了解群里的氛围和规则。提问前先搜索聊天记录，很可能你的问题已经有人问过。

> **常见问题**：群里消息太多，看不过来怎么办？
> 
> 建议：1）屏蔽群消息，每天固定时间翻看；2）利用微信的"搜一搜"功能，直接搜索关键词；3）把群里的精华内容收藏或转发到"文件传输助手"保存。

## 4. 学习路径推荐

不同的人有不同的背景和目标，因此需要不同的学习路径。下面为你设计了四条路径，请对号入座。

### 4.1 路径一：零基础快速入门（1-3 天）

**适合人群：**
- 完全不懂编程的小白
- 想快速体验 Hermes 的能力
- 时间有限，想先有个感性认识

**学习目标：**
- 成功安装并运行 Hermes Agent
- 体验 3-5 个内置 Skill
- 了解 Hermes 能做什么、不能做什么

**学习清单：**

```
Day 1：安装与初体验
├─ [30 分钟] 阅读官方 README，了解 Hermes 是什么
├─ [2 小时] 按照官方文档完成安装（Docker 方式推荐）
├─ [30 分钟] 配置 API Key，测试第一个对话
└─ [1 小时] 尝试 3 个内置 Skill（搜索、计算、文件读取）

Day 2：探索与玩耍
├─ [1 小时] 去 Skills Hub 下载 2-3 个感兴趣的社区 Skill
├─ [2 小时] 跟着一个教程视频，完成一个小项目
└─ [1 小时] 在社区群里看别人在讨论什么，了解使用场景

Day 3：总结与规划
├─ [1 小时] 整理你学到的内容，写一个学习笔记
├─ [1 小时] 思考：Hermes 能在你的工作/生活中解决什么问题？
└─ [1 小时] 制定下一步学习计划
```

**推荐资源：**
- 橙皮书入门篇
- B站"Hermes 安装教程"类视频
- 官方 Quick Start 文档

> **Tips**：这个阶段的目标是"跑起来"，不要纠结原理。安装过程中遇到问题很正常，去群里问或者搜 Issue，90% 的问题都有人遇到过。

### 4.2 路径二：系统性学习（1-2 周）

**适合人群：**
- 有一定技术基础（会一点 Python 或用过命令行）
- 想系统掌握 Hermes 的使用和开发
- 目标是能独立开发 Skill 或工作流

**学习目标：**
- 深入理解 Hermes 的核心架构
- 掌握 Skill 的开发和调试
- 能独立完成一个中等复杂度的项目

**学习清单：**

```
Week 1：核心概念与基础开发

Day 1-2：架构理解
├─ 阅读核心架构文档
├─ 理解 Agent、Skill、Tool、Memory 的关系
└─ 了解配置文件的各个参数含义

Day 3-4：Skill 开发入门
├─ 学习 Skill 的文件结构和语法
├─ 跟着教程写第一个简单 Skill
└─ 学习调试技巧（日志、断点）

Day 5-6：工具集成
├─ 了解内置工具的种类和用法
├─ 学习如何调用外部 API
└─ 实践：做一个调用天气 API 的 Skill

Day 7：休息 + 复盘

Week 2：进阶与实战

Day 8-9：记忆系统
├─ 学习三层记忆系统的原理
├─ 配置和使用记忆功能
└─ 实践：做一个能记住用户偏好的 Skill

Day 10-11：工作流编排
├─ 学习多步骤任务的编排
├─ 学习条件判断和错误处理
└─ 实践：做一个多步骤的数据处理工作流

Day 12-13：实战项目
├─ 选定一个项目目标（如：个人知识助手）
├─ 从 0 到 1 完成开发
└─ 测试、优化、写文档

Day 14：总结与分享
├─ 整理项目经验
├─ 写一篇教程或录一个视频
└─ 在社区分享你的项目
```

**推荐资源：**
- 官方完整文档
- 橙皮书进阶篇
- GitHub 上的示例项目
- 社区高手的 Skill 源码（最好的学习材料）

> **Tips**：系统性学习最忌讳"只看不练"。每学一个概念，立刻动手实践。哪怕只是改改参数、试试不同的配置，也比单纯阅读效果好 10 倍。

### 4.3 路径三：深度技术研究（1 个月）

**适合人群：**
- 开发者，想深入理解 Hermes 的底层实现
- 有贡献开源项目的意愿
- 目标是成为社区核心贡献者

**学习目标：**
- 理解 Hermes 的源码架构
- 掌握自定义工具的开发
- 能为社区贡献代码或 Skill

**学习清单：**

```
Week 1：源码阅读
├─ Clone 官方仓库
├─ 阅读项目结构和主要模块
├─ 从入口开始， tracing 一次完整的请求流程
└─ 理解核心类的职责和交互

Week 2：工具开发
├─ 学习 Tool 的接口规范
├─ 开发一个自定义 Tool（如连接某个特定 API）
├─ 学习测试方法，写单元测试
└─ 提交 PR 到社区（如果被合并，成就感爆棚）

Week 3：高级主题
├─ 自定义模型接入（本地模型、私有模型）
├─ 性能优化（并发、缓存、成本控制）
├─ 安全与权限管理
└─ 企业级部署方案

Week 4：贡献与输出
├─ 选择一个社区 Issue 解决
├─ 或开发一个高质量的 Skill 提交到 Skills Hub
├─ 写一篇深度技术文章
└─ 参与社区讨论，建立影响力
```

**推荐资源：**
- 源码本身（最好的老师）
- 官方架构文档
- 社区的技术讨论帖
- 相关论文（如 ReAct、CoT 等 Agent 相关论文）

> **Tips**：读源码时，不要试图一次性理解所有代码。先抓主干、看流程，再逐步深入细节。善用 IDE 的"跳转到定义"和"查找引用"功能。

### 4.4 路径四：从 OpenClaw 迁移（1-3 天）

**适合人群：**
- 之前用过 OpenClaw 或其他类似框架
- 想迁移到 Hermes，但不知道怎么做

**学习目标：**
- 理解 Hermes 和 OpenClaw 的异同
- 完成项目迁移
- 掌握 Hermes 特有的优势功能

**学习清单：**

```
Day 1：对比与评估
├─ 阅读 Hermes vs OpenClaw 的对比文章
```

**Hermes vs OpenClaw 核心差异：**

| 维度 | OpenClaw | Hermes |
|------|---------|--------|
| 安装复杂度 | 较复杂 | 简单（Docker 一键部署） |
| 配置文件 | 较繁琐 | 更简洁直观 |
| Skill 生态 | 较小 | 更活跃，Skills Hub 丰富 |
| 记忆系统 | 基础 | 三层记忆，更强大 |
| 社区活跃度 | 一般 | 更活跃 |
| 中文支持 | 较弱 | 更好 |

> **Tips**：迁移时不要追求"一次性完美迁移"。先让核心功能跑起来，再逐步优化。迁移过程中遇到 API 差异，多在社区搜索或提问。

## 5. 社区参与指南

### 5.1 如何提 Issue

GitHub Issue 是报告 Bug、请求新功能、提出问题的主要渠道。

**提 Issue 的正确姿势：**

```markdown
# Bug 报告模板

## 问题描述
[清楚描述你遇到的问题]

## 复现步骤
1. [第一步]
2. [第二步]
3. [第三步]

## 期望结果
[你希望发生什么]

## 实际结果
[实际发生了什么，附上完整的错误信息]

## 环境信息
- Hermes 版本：[如 1.2.3]
- 操作系统：[如 macOS 14.1]
- Python 版本：[如 3.11.4]
- 模型：[如 GPT-4]

## 附加信息
[截图、日志文件、相关配置等]
```

> **Tips**：提 Issue 前先搜索一下，确认没有重复的问题。一个好的 Issue 应该让维护者能快速理解并复现问题——信息越完整，解决速度越快。

> **常见问题**：我提的 Issue 没人回复怎么办？
> 
> 开源维护者都是志愿者，有自己的本职工作。通常：
> - Bug 报告：1-7 天内会有回复
> - 功能请求：可能需要几周甚至更久
> - 使用问题：建议在 Discussions 板块提问，而不是开 Issue
> 
> 如果 Issue 长时间没回复，可以礼貌地 Bump 一下（在评论里补充新信息或询问进展）。

### 5.2 如何提交 PR

Pull Request（PR）是向项目贡献代码的方式。

**PR 流程：**

```bash
# 1. Fork 官方仓库到自己的账号
# 在 GitHub 页面上点击 Fork 按钮

# 2. Clone 你 Fork 的仓库
git clone https://github.com/你的用户名/hermes-agent.git
cd hermes-agent

# 3. 创建新分支
git checkout -b fix-some-bug

# 4. 修改代码
# ... 编辑文件 ...

# 5. 提交更改
git add .
git commit -m "fix: 修复了 XXX 问题"

# 6. 推送到你的仓库
git push origin fix-some-bug

# 7. 在 GitHub 上发起 Pull Request
```

**PR 注意事项：**
- 每个 PR 只解决一个问题，不要混多个改动
- 写清楚 PR 描述，说明改了什么、为什么改
- 确保代码通过了测试
- 遵守项目的代码风格（通常有 lint 检查）
- 对维护者的 review 意见保持开放态度

> **Tips**：第一次贡献可以从简单的开始，比如：
> - 修复文档中的错别字
> - 补充某个函数的注释
> - 增加一个简单的测试用例
> 
> 小 PR 更容易被快速合并，也能帮你熟悉流程。

### 5.3 如何贡献技能

贡献 Skill 是参与 Hermes 社区最轻松也最有价值的方式之一。

**Skill 贡献流程：**

```
1. 开发你的 Skill
   ├─ 确保 Skill 功能完整、测试通过
   ├─ 写好 README，包含使用说明和示例
   └─ 遵守 Skill 命名和格式规范

2. 上传到 Skills Hub
   ├─ 注册 Skills Hub 账号
   ├─ 填写 Skill 信息（名称、描述、标签等）
   ├─ 上传 Skill 文件
   └─ 提交审核

3. 推广你的 Skill
   ├─ 在社区分享你的 Skill
   ├─ 写一篇使用教程
   └─ 收集用户反馈，持续迭代
```

**高质量 Skill 的评判标准：**
- 解决真实问题
- 开箱即用，配置简单
- 文档完整，示例丰富
- 错误处理完善
- 代码规范，易于维护

### 5.4 如何参与文档翻译

文档翻译是帮助中文社区发展的绝佳方式，而且门槛很低——不需要会编程，只要会中英双语即可。

**参与方式：**
1. 去中文文档站（如 hermes-doc.aigc.green）查看如何参与
2. 或者直接去 GitHub 文档仓库找 "help wanted" 或 "translation" 标签的 Issue
3. 认领一页或一节，翻译后提交 PR

**翻译注意事项：**
- 保持技术术语的准确性
- 中文表达要通顺自然，不要直译
- 代码块和命令不需要翻译
- 保持原文的格式和结构

> **Tips**：翻译过程中，如果遇到不确定的术语，可以去社区群里问问大家的习惯译法。统一术语翻译对文档质量非常重要。

### 5.5 社区行为准则

参与社区时，请遵守以下基本准则：

**要做的（Do's）：**
- 友好、尊重地与其他成员交流
- 分享你的知识和经验
- 对他人的贡献表示感谢
- 在提问前先搜索和尝试
- 提供具体、完整的信息

**不要做的（Don'ts）：**
- 不要人身攻击或嘲讽他人
- 不要发广告或垃圾信息
- 不要重复提问（尤其是在短时间内）
- 不要泄露他人的隐私信息
- 不要要求维护者"立刻"解决你的问题

> **Tips**：社区是一个"给予"的地方。你帮助别人的越多，别人帮助你的也越多。即使是新手，也可以回答比自己更新手的问题——教学相长。

## 7. 常见问题总汇（FAQ）

本章整理了 25 个最常见的问题，按主题分类。如果你遇到了问题，先来这里找找答案。

### 7.1 安装相关（5 问）

**Q1：安装时提示 "Docker 未启动" 怎么办？**

A：请确保 Docker Desktop（Mac/Windows）或 Docker Daemon（Linux）正在运行。
```bash
# 检查 Docker 状态
docker info

# 如果未启动，手动启动
# Mac/Windows: 打开 Docker Desktop 应用
# Linux: sudo systemctl start docker
```

**Q2：Python 版本有什么要求？**

A：Hermes 需要 Python 3.10 或更高版本。
```bash
# 检查 Python 版本
python --version

# 如果版本不够，建议用 pyenv 或 conda 安装新版本
pyenv install 3.11.4
pyenv global 3.11.4
```

**Q3：安装依赖时提示 "权限不足" 怎么办？**

A：不要用 sudo 安装 Python 包，建议使用虚拟环境：
```bash
# 创建虚拟环境
python -m venv hermes-env

# 激活虚拟环境
source hermes-env/bin/activate  # Linux/Mac
# 或
hermes-env\Scripts\activate  # Windows

# 再安装依赖
pip install -r requirements.txt
```

**Q4：Windows 上安装有什么特别注意的吗？**

A：Windows 用户常见问题：
- 建议使用 WSL2（Windows Subsystem for Linux）安装，体验更接近 Linux
- 如果不用 WSL，确保安装了 Visual C++ Build Tools
- 路径中避免中文和空格

**Q5：安装完成后如何验证是否成功？**

A：运行以下命令：
```bash
hermes --version
hermes doctor  # 检查环境和配置
```
如果都能正常输出，说明安装成功。

### 7.2 使用相关（5 问）

**Q6：如何修改配置文件？**

A：配置文件通常位于 `~/.config/hermes/config.yaml`（Linux/Mac）或 `%USERPROFILE%\.config\hermes\config.yaml`（Windows）。

```bash
# 用默认编辑器打开配置文件
hermes config edit

# 或手动找到并编辑
nano ~/.config/hermes/config.yaml
```

**Q7：Agent 回复很慢怎么办？**

A：可能的原因和解决方案：
1. **网络问题**：检查你的网络连接，尤其是访问 API 的延迟
2. **模型选择**：GPT-4 比 GPT-3.5 慢但质量高，根据场景选择
3. **工具调用太多**：减少单次请求的工具调用数量
4. **本地模型**：如果用本地模型，检查硬件资源（GPU/内存）

**Q8：如何查看运行日志？**

A：
```bash
# 实时查看日志
hermes logs -f

# 查看最近 100 行
hermes logs --tail 100

# 查看某个 Skill 的日志
hermes logs --skill skill-name
```

**Q9：如何备份和迁移我的配置？**

A：Hermes 的配置和记忆数据都在配置目录中：
```bash
# 备份
cp -r ~/.config/hermes ~/hermes-backup

# 迁移到新机器
cp -r ~/hermes-backup ~/.config/hermes
```

**Q10：如何停止正在运行的任务？**

A：
```bash
# 查看运行中的任务
hermes task list

# 停止指定任务
hermes task stop <task-id>

# 停止所有任务
hermes task stop-all
```

### 7.3 模型相关（5 问）

**Q11：支持哪些模型？**

A：Hermes 支持多种模型：
- **OpenAI**：GPT-4、GPT-4o、GPT-3.5-turbo
- **Anthropic**：Claude 3/3.5 系列
- **DeepSeek**：DeepSeek-V2/V3
- **本地模型**：通过 Ollama、vLLM 等接入
- **其他**：任何兼容 OpenAI API 格式的服务

**Q12：API Key 怎么申请？**

A：
- **OpenAI**：https://platform.openai.com/api-keys
- **Anthropic**：https://console.anthropic.com/
- **DeepSeek**：https://platform.deepseek.com/

> **注意**：API Key 是收费服务，请妥善保管，不要在公开场合泄露。

**Q13：使用本地模型怎么配置？**

A：以 Ollama 为例：
```yaml
# config.yaml
model:
  provider: ollama
  base_url: "http://localhost:11434"
  model: "llama3.1:8b"
```

**Q14：API 调用费用大概多少？**

A：取决于你的使用量和选择的模型：
- GPT-3.5：约 0.002 美元/1K tokens
- GPT-4：约 0.03 美元/1K tokens
- Claude 3.5 Sonnet：约 0.003 美元/1K tokens
- 本地模型：电费成本，几乎为零

一个普通用户轻度使用，每月 API 费用通常在 10-50 元人民币。

**Q15：如何选择合适的模型？**

A：选择建议：
- **日常对话/简单任务**：GPT-3.5 或 DeepSeek（便宜且够用）
- **复杂推理/代码任务**：GPT-4o 或 Claude 3.5 Sonnet
- **隐私敏感场景**：本地模型（Llama、Qwen 等）
- **追求性价比**：DeepSeek-V3、Claude 3 Haiku

### 7.4 技能相关（5 问）

**Q16：Skill 文件放在哪里？**

A：
```bash
# 用户级 Skill 目录
~/.config/hermes/skills/

# 项目级 Skill 目录
./skills/  # 项目根目录下
```

**Q17：如何调试 Skill？**

A：
```bash
# 以调试模式运行 Skill
hermes run skill:my-skill --debug

# 查看详细日志
hermes run skill:my-skill --verbose

# 逐步执行（交互式）
hermes run skill:my-skill --step
```

**Q18：Skill 里怎么引用环境变量？**

A：
```yaml
# 在 Skill 文件中使用环境变量
api_key: "${MY_API_KEY}"
base_url: "${API_BASE_URL|https://default.example.com}"  # 带默认值
```

**Q19：Skill 执行出错如何排查？**

A：排查步骤：
1. 检查 Skill 语法（YAML 缩进是否正确）
2. 查看错误日志，定位出错的步骤
3. 检查工具参数是否正确
4. 确认 API Key 和权限有效
5. 在 GitHub Issues 搜索类似错误

**Q20：如何分享我的 Skill 给别人？**

A：
```bash
# 导出 Skill
hermes skill export my-skill -o ./my-skill.zip

# 对方导入
hermes skill import ./my-skill.zip

# 或上传到 Skills Hub，对方直接安装
hermes skill install 你的用户名/my-skill
```

### 7.5 其他（5 问）

**Q21：Hermes 和 LangChain 有什么区别？**

A：
| 维度 | Hermes | LangChain |
|------|--------|-----------|
| 定位 | 面向终端用户的 Agent 框架 | 面向开发者的 LLM 应用框架 |
| 使用难度 | 较低，配置驱动 | 较高，需要编程 |
| 目标用户 | 个人用户、小型团队 | 开发者、企业 |
| Skill 生态 | 内置 Skill 商店 | 主要通过代码集成 |
| 部署方式 | 本地/服务器均可 | 主要在应用中集成 |

简单说：**如果你是想"用"Agent，选 Hermes；如果你是想"开发"Agent 应用，两者都可以。**

**Q22：Hermes 是免费的吗？**

A：Hermes 本身是**开源免费**的（MIT 协议）。但使用过程中可能产生：
- API 调用费用（付给模型提供商）
- 服务器费用（如果你部署在云上）
- 部分第三方 Skill 可能收费

**Q23：如何更新 Hermes 到最新版本？**

A：
```bash
# 如果是 pip 安装
pip install -U hermes-agent

# 如果是 Docker 部署
docker pull nousresearch/hermes-agent:latest
```

> **注意**：升级前请备份配置文件，并阅读 Release Notes。

**Q24：遇到问题去哪求助？**

A：按优先级：
1. **官方文档**：最权威
2. **GitHub Issues**：搜索是否有人遇到过
3. **Discord/微信群**：实时交流
4. **知乎/B站**：可能有教程
5. **直接提问**：在 Discord 或 GitHub Discussions 提问

**Q25：我想为 Hermes 做贡献，但不知道从何开始？**

A：建议路径：
1. 先用 Hermes 做几个项目，熟悉它
2. 去 GitHub Issues 找标有 "good first issue" 的问题
3. 从文档改进、Bug 修复等小贡献开始
4. 逐步过渡到功能开发和 Skill 贡献
5. 积极参与社区讨论，建立影响力
