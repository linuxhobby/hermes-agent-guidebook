## 1. 为什么写这本白皮书

### 1.1 AI Agent 时代已来，但大多数人还在门外观望

2026 年的春天，如果你打开任何一个科技媒体的首页，"AI Agent" 这个词几乎会占据半壁江山。

有人在展示自己的 Agent 如何自动完成一份行业研究报告——从数据收集、竞品分析到 PPT 生成，全程无需人工干预。有人在炫耀自己的 Agent 同时运营着 20 个自媒体账号，每天自动选题、写作、排版、发布，月收入超过了很多上班族的年薪。还有人在企业里部署了智能客服 Agent，不仅 7×24 小时在线回答问题，还能主动学习新的产品知识，越用越聪明。

但与此同时，我也看到了另一群人的困惑。

他们可能是你，也可能是你的朋友——听说过 ChatGPT 能联网查资料，但不知道为什么有人能让 AI "自动" 做那么多事情；收藏了几十个关于 Agent 的视频和文章，但看完还是觉得云里雾里；下载过 AutoGPT 或 OpenClaw，但运行起来不是报错就是卡在原地，最后无奈放弃。

这像极了二十年前互联网刚普及时的情景。那时候也有人能建网站、写博客，但更多人面对 "HTML 是什么" 这个问题时一脸茫然。区别在于，当年你不懂互联网，最多是错过了一个信息获取的便利渠道；而在 2026 年，不懂 AI Agent，可能意味着你将在效率竞赛中被远远甩在后面。

> **Tips**
>
> 不要觉得 "AI Agent 是程序员的事"。2026 年的 Agent 工具已经发展到了"半代码化"甚至"无代码化"的阶段。你不需要会写 Python，也能搭建一个属于自己的智能助手。这本书的目标，就是让你——无论你是否有技术背景——都能真正 "入门" 并 "用好" Hermes Agent。

### 1.2 市面上的教程碎片化严重，缺乏系统性

让我问你一个问题：如果你想学 Hermes Agent，现在能找到哪些学习资源？

首先，你可以去 GitHub 上看官方 README。写得不错，但毕竟是面向开发者的技术文档，充满了配置参数和命令行指令。对于没有技术背景的人来说，读完第一段可能就放弃了。

然后，你可以去 B站、YouTube 搜视频教程。有些视频确实讲得很精彩，但问题是——它们大多是碎片化的。这个 UP 主讲安装，那个博主讲配置，还有一个视频专门介绍某个技能。你学完了 "如何安装"，但 "为什么要这样配置" 还是不清楚；你看懂了 "这个技能怎么用"，但 "如何创建自己的技能" 依然是个谜。

更麻烦的是，很多教程的时效性很差。AI Agent 这个领域发展太快了，三个月前的 "最佳实践" 可能现在已经过时了。你按照某个教程操作，结果因为版本更新，命令已经变了、接口已经改了，折腾半天还是跑不通。

> **常见问题**
>
> **Q：我已经看过很多关于 OpenClaw 的教程了，为什么还要学 Hermes？**
>
> **A：** 好问题！Hermes 和 OpenClaw 确实有很多相似之处（毕竟它们有共同的起源），但 Hermes 在架构设计、记忆系统、技能生态等方面都有重大升级。如果你只懂 OpenClaw，那么你在使用 Hermes 时会错过很多强大的新功能。更重要的是，本书不仅教你 "怎么用"，更教你 "为什么这样设计"，这才是系统性学习的价值。

### 1.3 从"养虾"到"养马"的转变，需要一本完整指南

让我用一个类比来解释这个变化。

假设你是一个喜欢养宠物的人。最开始，你养的是虾——投点饵料，换换水，虾就能活。对应到 AI 世界，这就是早期的 ChatGPT 时代：你给 AI 一个问题，它给你一个答案。简单直接，但互动是单向的，AI 不会"记住"你们之前的对话，也不会"主动"为你做什么。

后来，你开始养鱼。鱼缸需要更复杂的生态系统——过滤、打氧、温控、光照。对应到 AI 世界，这就是 RAG（检索增强生成）和工具调用时代：AI 可以查资料、调用工具，但它还是按照你的指令一步一步执行，没有自主性。

再后来，你开始养猫养狗。它们有自己的性格、记忆和习惯。它们会在你回家时迎接你，会记住你不喜欢什么，甚至会自己学会一些"技能"——比如开门、按铃要食物。对应到 AI 世界，这就是 Agent 时代：AI 不再只是回答问题，而是能够自主感知环境、做出决策、执行任务，并且从经验中学习和进化。

现在，你决定养一匹马。

马和猫狗又不一样。马可以载着你走很远的路，可以帮你完成真正的"工作"——运输、农耕、甚至比赛。但养马也更需要系统性知识：如何选择品种、如何训练、如何建立信任关系、如何让它理解你的指令、如何让它越骑越顺手。

Hermes Agent，就是这样一匹马。

它不是玩具，不是宠物，而是一个真正能帮你干活的伙伴。但要把这匹马"养好"、"训好"、"用好"，你需要一本完整的指南——从认识马、选马、养马，到骑马、赛马、甚至培育自己的马。这就是我写这本白皮书的初衷。

> **注意**
>
> "养马"这个比喻贯穿了整本书。你会发现，很多复杂的概念我都会用养马来类比。Hermes 这个名字本身就来自希腊神话中的信使之神——他骑着飞马穿梭天地，传递信息。所以，"养马"不仅是类比，也是一种隐喻：你正在培养一个快速、聪明、忠诚的 AI 伙伴。

### 1.4 本书的定位：不是技术论文，是你的 AI 伙伴养成手册

在动笔之前，我给自己定了一个硬性要求：**这本书不能让任何一个读者感到"被排斥"。**

很多技术文档都有一个隐含的假设：读者已经具备了某些基础知识。比如，它们会默认你知道什么是 API、什么是 JSON、什么是环境变量。但对于真正的初学者来说，这些"基础知识"本身就是一道门槛。

所以，这本书的定位非常明确：

- **它不是学术论文**。你不需要有计算机科学学位才能读懂。
- **它不是 API 文档**。我不会罗列每一个函数的参数和返回值。
- **它不是一本快餐式的操作手册**。我不满足于只告诉你"点这里、填那里"。
- **它是一本养成手册**。我会带你从"完全不懂"走到"真正会用"，再到"能够根据自己的需求定制"。

书中的每一个概念，我都会用生活化的类比来解释。每一个操作步骤，我都会假设你是第一次接触。每一个可能出现的问题，我都会提前告诉你怎么解决。

> **Tips**
>
> 如果你在某个章节遇到了不理解的术语，不要慌。翻到本章最后的"术语表"，或者在全书最后的索引中查找。我承诺：书中出现的每一个专业术语，至少会在第一次出现时给出通俗解释。

### 1.5 这本书的独特之处

市面上的技术书籍很多，这本书有什么不同？

**第一，它是中文的。** 我知道这听起来有点奇怪——但事实是，AI Agent 领域最好的资料大多是英文的。官方文档是英文的，GitHub 上的讨论是英文的，Discord 里的交流也是英文的。对于英文不好的朋友来说，这无疑是一道巨大的门槛。本书全程使用中文写作，所有的命令、配置和界面截图（如果有的话）都会配合中文解释。

**第二，它是实战导向的。** 我不会只讲理论。每一章都有动手环节，每一个概念都有对应的操作步骤。读完第一章，你就能说出 Hermes 是什么；读完第五章，你就能在自己的电脑上跑起来一个 Hermes Agent；读完第十二章，你就能让它自动完成真正的工作任务。

**第三，它覆盖了从入门到精通的完整路径。** 全书 16 册，从"什么是 AI Agent"这种基础问题，到"如何设计一个企业级多 Agent 系统"这种高阶话题，全部覆盖。你不需要再去其他地方找资料——这本书就是一本"全栈"指南。

**第四，它关注"为什么"，而不仅是"怎么做"。** 知其然，更要知其所以然。我不会只告诉你"配置文件里这个参数要填 true"，我会解释"为什么填 true 会更好"。理解了设计背后的逻辑，你才能真正灵活运用，而不是机械地照搬。

**第五，它持续更新。** AI Agent 领域变化太快了，一本写死在纸上的书很快就会过时。本书采用数字出版形式，会随着 Hermes 的版本迭代持续更新。你手中的永远是最新的内容。

## 2. Hermes Agent 是什么——一句话说清楚

### 2.1 用最通俗的语言解释 Hermes Agent

好，现在让我们来解决一个根本问题：**Hermes Agent 到底是什么？**

如果你让我用一句话回答，我会说：

> **Hermes Agent 是一个生活在你的电脑里的、会学习、会进化的 AI 助手。它能理解你的需求，自己想办法完成任务，并且越用越懂你。**

让我再展开一点。

想象你有一个私人助理。这个助理非常聪明，会写文章、会查资料、会写代码、会发邮件。但不仅如此，他还有以下几个特点：

1. **他有超强的记忆力**。你三年前随口提过的一个偏好，他现在还记得。你们上个月的某次对话，他能准确回忆起来。
2. **他会自己学新技能**。你从来没教过他怎么操作 Excel 宏，但他看你做了几次之后，自己就学会了。下次你需要的时候，他直接就能上手。
3. **他会主动思考**。你让他"关注一下竞争对手的动向"，他不会傻等你的下一步指令，而是自己去搜索、分析、总结，然后把一份报告放到你桌上。
4. **他会不断进化**。刚来的时候他还比较"青涩"，但跟着你工作了几个月之后，他对你的行业、你的工作习惯、你的说话方式都越来越熟悉，做事越来越得心应手。

Hermes Agent，就是这样一个"数字版"的私人助理。

但比私人助理更强大的是，它：

- **不需要工资**，只需要你的电脑或服务器能跑起来
- **不需要睡觉**，可以 7×24 小时工作
- **不会跳槽**，你的数据和记忆永远属于你
- **可以复制**，你可以训练好一个 Agent，然后复制给团队里的其他人用

> **Tips**
>
> 很多新手容易把 Agent 想象成一个"更聪明的 ChatGPT"。这个理解不够准确。ChatGPT 是一个"对话系统"——你问一句，它答一句。而 Agent 是一个"行动系统"——你给它一个目标，它自己规划步骤、调用工具、执行任务，最后把结果交给你。ChatGPT 是"你推一下它动一下"，Agent 是"你给方向它自己跑"。

### 2.2 它和 ChatGPT、Claude 等产品的区别

为了让你更清楚地理解 Hermes 的定位，我们来做个横向对比。

| 特性 | ChatGPT / Claude | Hermes Agent |
|------|------------------|--------------|
| 运行位置 | 云端（OpenAI/Anthropic 的服务器） | 你的本地电脑或私有服务器 |
| 交互方式 | 一问一答的对话 | 目标驱动的任务执行 |
| 记忆能力 | 有限上下文（通常几轮对话） | 三层记忆系统，跨会话持久化 |
| 工具使用 | 内置工具（联网、代码解释器等） | 47+ 内置工具 + 6000+ MCP 服务 |
| 学习进化 | 模型本身不学习，靠更新迭代 | 运行时自学习，技能自我进化 |
| 数据隐私 | 数据上传到云端 | 数据完全本地保留 |
| 定制化程度 | 有限（提示词、自定义 GPT） | 极高（技能、记忆、工具全可定制） |
| 成本 | 按调用付费（订阅制） | 本地运行免费，API 调用按需付费 |

这个表格的信息量很大，让我挑几个关键点来解释。

**关于运行位置**：ChatGPT 和 Claude 是"云服务"。你每说一句话，数据都要发到 OpenAI 或 Anthropic 的服务器上处理。这意味着你的对话内容理论上可以被服务商看到（虽然他们承诺不用于训练，但你很难完全验证）。Hermes 是"本地服务"，所有的数据都在你自己的机器上处理。对于处理敏感信息（比如公司内部的商业数据、个人隐私信息）来说，这是一个巨大的优势。

**关于交互方式**：ChatGPT 是"你问我答"。你说"帮我写个邮件"，它给你写一封邮件。你说"再改得正式一点"，它再改。你需要一步一步引导它。Hermes 是"你给目标，它自己想办法"。你说"帮我给张总发一封感谢邮件，感谢他在上周会议中的建议，顺便提一下我们下周的方案"，Hermes 会自动完成：写邮件、检查语气、调用邮件发送工具——整个过程你不需要干预。

**关于学习进化**：这是最核心的区别。ChatGPT 不会因为你经常用它就变得更懂你。它"懂"什么，完全取决于 OpenAI 训练了什么、更新了什么。而 Hermes 有一个自学习闭环：你每次跟它交互，它都会分析这次交互，提取有价值的经验，存到记忆里。下次遇到类似情况，它会基于这些经验做出更好的决策。这就是"越用越聪明"的真正含义。

> **注意**
>
> 这并不意味着 Hermes 比 ChatGPT "更好"。它们是为不同场景设计的工具。如果你只是偶尔问几个问题、聊聊天，ChatGPT 可能更方便（不需要安装、随时随地可用）。但如果你是每天都要跟 AI 协作完成大量任务的重度用户，或者你需要处理敏感数据，Hermes 的本地化和自主性就是无可替代的优势。

### 2.3 它和 AutoGPT、MetaGPT 等框架的区别

Hermes 属于"AI Agent 框架"这个大家族。在这个家族里，还有很多知名的成员：AutoGPT、BabyAGI、MetaGPT、CrewAI、AutoGen、LangGraph 等等。Hermes 和它们有什么不同？

**AutoGPT**：这是 Agent 领域的"开山鼻祖"之一。它的核心思想是给 AI 一个目标，然后让 AI 自己分解任务、执行、反思、再执行，循环往复直到完成。AutoGPT 开创了很多 Agent 设计的先河，但它的问题也很明显：容易陷入"思考循环"（反复思考但不动手）、任务分解质量不稳定、缺乏有效的记忆管理机制。

**MetaGPT**：这个框架的灵感来自软件公司的组织架构。它模拟了一个完整的软件开发团队——产品经理写 PRD、架构师设计系统、工程师写代码、测试员测 Bug。多个 AI 角色协作完成一个复杂任务。这个想法很酷，但它主要面向软件开发场景，对于非开发任务（比如投研、内容创作、客服）的适配性有限。

**CrewAI**：这是一个更轻量级的多 Agent 框架。你可以快速组建一个"团队"，给每个成员分配角色和任务。它的优点是简单上手快，缺点是功能相对基础，缺乏深度定制和自学习能力。

**LangGraph**：这是 LangChain 团队推出的 Agent 编排框架。它用"图"的方式来设计 Agent 的工作流程，非常适合构建复杂、确定性的业务流程。它的学习曲线比较陡峭，更适合有 LangChain 经验的开发者。

**OpenClaw**：这是 Hermes 的"前辈"，也是 Nous Research 团队之前的作品。OpenClaw 在 2025 年达到了 309k Stars 的惊人成绩，是 GitHub 上最受欢迎的 Agent 项目之一。但它在架构上存在一些局限性——尤其是在记忆管理和技能进化方面——这些局限性促使了 Hermes 的诞生。

**Hermes Agent**：Hermes 继承了 OpenClaw 的优秀基因，但进行了全面的架构升级。它的核心优势在于：

1. **三层记忆系统**：短期记忆（会话上下文）+ 持久记忆（SQLite 全文检索）+ Skill 级记忆（跨会话知识沉淀）。这让 Hermes 拥有了真正的"长期记忆"能力。
2. **自主技能创建**：Hermes 不仅能执行预定义的技能，还能在运行过程中自动创建新技能，并让这些技能自我进化。
3. **按需检索**：不是把所有记忆都塞进上下文，而是根据当前任务的需求，智能地从记忆库中检索相关信息。
4. **用户建模**：Hermes 会为每个用户建立独立的画像模型，记录你的偏好、习惯、工作模式，从而实现真正的"个性化"。

> **常见问题**
>
> **Q：这些框架我能同时用吗？还是只能选一个？**
>
> **A：** 实际上，它们并不是完全互斥的。比如，你可以在 Hermes 中集成 LangGraph 来做流程编排，也可以用 CrewAI 的多 Agent 协作模式来扩展 Hermes 的能力。但在入门阶段，我建议你先专注学好一个框架。Hermes 的功能已经非常全面了，足以覆盖绝大多数场景。

### 2.4 核心概念：会自我学习、自我进化的本地 AI 助手

让我们把 Hermes 的核心特点再浓缩一下，归纳为四个关键词：

**本地（Local）**

Hermes 运行在你的机器上。你的对话记录、生成的文件、积累的记忆，全部存储在你的本地磁盘上。你不需要担心数据被上传到云端，不需要担心服务商的政策变化影响你的使用，不需要担心网络波动导致服务中断。

对于有数据隐私要求的企业来说，这是刚需。对于个人用户来说，这也意味着你对系统有完全的控制权。

**自主（Autonomous）**

传统的 AI 工具是"被动"的——你让它做什么，它才做什么。Hermes 是"主动"的——你给出一个目标，它会自己规划步骤、调用工具、处理异常、调整策略，直到目标达成。

比如，你说"帮我准备明天跟李总的会议"，Hermes 会自动：
- 从你的日历里找到明天跟李总的会议安排
- 检索你和李总的历史往来记录
- 查找李总所在公司的最新动态
- 整理你们上次会议的待办事项
- 生成一份会议准备清单和谈话要点
- 把这些内容整理成一份文档发给你

整个过程，你只需要说一句话。

**学习（Learning）**

Hermes 有一个核心的自学习闭环。每次交互结束后，系统会分析：
- 这次任务完成得怎么样？
- 用户的反馈是什么？（满意/不满意/需要修改）
- 有哪些经验可以沉淀下来？
- 有没有新的模式需要记录？

这些分析结果会被存入记忆系统，成为 Hermes 的"经验"。下次遇到类似情况，它会基于这些经验做出更好的决策。

**进化（Evolution）**

学习是"积累知识"，进化是"升级能力"。Hermes 的技能系统支持自我进化：
- 它能根据你的使用模式，自动创建新的技能
- 它能分析技能的使用效果，自动优化技能的实现
- 它能从社区共享的技能库中学习，吸收最佳实践

这就好比你的助理不仅记住了你教他的东西，还能自己琢磨出更高效的工作方法，并且跟同行交流取长补短。

> **Tips**
>
> "自学习"和"自我进化"是 Hermes 最独特的能力，但也是新手最容易误解的概念。它不是说 Hermes 会变得像人类一样"聪明"，而是说它会变得越来越"适合你"。它不会突然学会你从来没提过的东西，但它会把跟你交互过程中积累的经验，转化成更高效、更精准的服务。

## 3. 为什么选择 Hermes 而不是其他框架

### 3.1 2026 年 AI Agent 市场格局概览

在做出选择之前，我们需要先看清整个市场的全貌。

2026 年的 AI Agent 市场，可以用"百花齐放、群雄并起"来形容。大致可以分为几个阵营：

**第一阵营：科技巨头的闭源产品**

OpenAI 的 GPT-4o with Tools、Anthropic 的 Claude with Computer Use、Google 的 Gemini Agents、Microsoft 的 Copilot Agents。这些产品的特点是：技术能力强、集成度高、生态丰富。但它们的缺点也同样明显：闭源（你无法知道内部怎么工作）、数据上云（隐私风险）、按量付费（成本不可控）、定制受限（只能按照厂商的设计使用）。

**第二阵营：开源框架**

这是开发者和极客的主战场。从早期的 AutoGPT、BabyAGI，到中期的 LangChain、LlamaIndex，再到近两年的 OpenClaw、Hermes、CrewAI、AutoGen、LangGraph 等，开源框架的数量和质量都在快速提升。2026 年，GitHub 上 Agent 相关的仓库数量已经超过了 50,000 个，Star 数超过 10k 的项目有 30 多个。

**第三阵营：垂直领域方案**

一些公司专注于特定领域的 Agent 解决方案。比如，有的专门做投研 Agent，有的专门做客服 Agent，有的专门做内容创作 Agent。这些方案在特定场景下做得很好，但通用性不足。

**第四阵营：低代码/无代码平台**

面向非技术用户的可视化搭建平台。用户通过拖拽组件、配置规则来构建简单的 Agent 工作流。门槛低，但灵活性和能力上限也低。

在这个格局中，Hermes 位于第二阵营——开源框架——的领先位置。

> **注意**
>
> 2026 年 4 月，Hermes Agent 在 GitHub 上的 Star 数已经达到了 97.8k，而它正式发布才两个月。这个增长速度在整个 AI 开源领域都是非常罕见的。相比之下，OpenClaw 用了约一年时间才达到类似的 Star 数。这说明了社区对 Hermes 的期待和认可。

### 3.2 Hermes 的五大核心优势

为什么 Hermes 能在这么短的时间内获得如此高的关注？我总结了五个核心优势。

#### 优势一：记忆自动归档

记忆是 Agent 智能化的根基。如果一个 Agent 每次跟你对话都像是"第一次见你"，那它跟 ChatGPT 有什么区别？

Hermes 的三层记忆系统解决了这个问题：

**第一层：会话记忆（短期上下文）**

就像人类的短期记忆一样，记录当前对话的上下文。你前面说了什么，Agent 后面还能记得。这一层确保了对话的连贯性。

**第二层：持久记忆（长期知识库）**

利用 SQLite 数据库配合 FTS5 全文检索引擎，Hermes 可以把所有重要的信息长期保存下来。你上个月提到的一个项目细节，今天问起来它还能准确回答。

**第三层：Skill 级记忆（跨会话知识沉淀）**

这是 Hermes 的独创。当你在多个会话中反复处理某类任务时，Hermes 会自动提炼出"通用模式"，形成 Skill 级别的记忆。这些记忆不仅服务于当前会话，还会被用于优化技能本身。

> **类比**
>
> 想象你有一个笔记本。第一层记忆像是你在谈话时随手记的便签——随时能看到，但空间有限。第二层记忆像是你的日记本——把重要的事情详细记录下来，需要的时候可以翻阅。第三层记忆像是你的"工作方法论"——你做了 100 个项目后总结出来的最佳实践，变成了你的"肌肉记忆"。

#### 优势二：自主创建 Skill

传统 Agent 框架的技能系统是"静态"的——开发者写好技能代码，用户拿来用。如果用户需要一个框架里没有的技能，就得等开发者更新，或者自己写代码。

Hermes 打破了这种模式。它具备**自动技能创建**的能力：

当你反复执行一类任务时，Hermes 会识别出这个模式。比如，你每周都要"收集竞品动态、整理成周报、发送给团队"。做了几周之后，Hermes 会主动问你："我发现你每周都在做类似的任务，是否需要我创建一个自动化的技能？"

如果你同意，Hermes 会自动：
1. 分析这个任务的步骤和依赖
2. 生成对应的 Skill 代码
3. 注册到技能库中
4. 下次你需要的时候，一键执行

更厉害的是，这个 Skill 不是一成不变的。随着你使用的次数增多，Hermes 会不断优化它——简化步骤、修正错误、增加新的功能。

> **Tips**
>
> 自动创建 Skill 的功能默认是开启的，但你可以在配置中关闭它，或者设置触发阈值（比如"至少重复 3 次才提示创建"）。对于喜欢手动控制一切的高级用户，这些选项给你了充分的自由度。

#### 优势三：技能自我进化

不仅新技能可以自动创建，现有的技能也会自我进化。

Hermes 有一个内置的**技能评估和优化循环**：

1. **执行监控**：每次技能执行时，系统会记录执行时间、成功率、用户反馈等指标。
2. **效果分析**：定期（默认每周）分析技能的使用数据，找出效率瓶颈和常见失败点。
3. **自动优化**：基于分析结果，尝试改进技能的实现。比如，把串行执行的步骤改为并行，或者增加错误处理的逻辑。
4. **A/B 测试**：新版本的技能不会直接替换旧版本，而是先在小部分场景中试用，确认效果更好后才全面推广。
5. **用户确认**：对于重大变更，Hermes 会征求你的同意。对于细微优化，它可以自动应用。

这个过程就像是你的助理不仅在做事情，还在"琢磨怎么把事情做得更好"。

> **常见问题**
>
> **Q：技能自我进化会不会把我的技能改坏？**
>
> **A：** 不会。Hermes 采用了多重保护机制。首先，所有优化都会先创建新版本，原版本会被保留。其次，新版本会经过 A/B 测试验证。第三，你可以随时回滚到旧版本。第四，你可以设置某些技能为"锁定"状态，禁止自动优化。安全性是设计的第一优先级。

#### 优势四：按需检索

大语言模型有一个固有的限制：**上下文窗口有限**。即使是最先进的模型，也只能同时处理几十万字的上下文。如果 Agent 把所有历史记忆都塞进上下文，很快就会爆表。

Hermes 的解决方案是**按需检索（Retrieval on Demand）**：

当接收到一个新任务时，Hermes 会先进行"需求分析"：这个任务需要什么知识？需要哪些历史背景？需要调用什么技能？然后，它只去检索相关的记忆片段，把不相关的信息过滤掉。

这就好比一个优秀的律师在接案子时，不会把律所所有的卷宗都翻一遍，而是根据案情快速定位到相关的判例和法规。

按需检索的核心技术是**语义检索 + 关键词检索的混合方案**：

- **语义检索**：利用向量嵌入技术，找到"意思相近"的内容。比如，你问"上次谈的那个合作"，即使你没有用"合作"这个词，系统也能根据语义理解找到相关内容。
- **关键词检索**：利用 FTS5 全文检索，快速定位包含特定关键词的内容。
- **混合排序**：把两种检索的结果合并，按相关度排序，选出最相关的片段送入上下文。

> **类比**
>
> 想象你的记忆是一座巨大的图书馆。传统的 Agent 每次借书都是把整本书搬走（把所有记忆塞进上下文）。Hermes 的做法是，先由一个聪明的图书管理员（检索系统）分析你需要什么，然后只把相关的那几页复印给你。这样效率更高，而且不会超重。

#### 优势五：用户建模

这是 Hermes 最具前瞻性的设计之一。

每个用户都是独特的。你的说话方式、工作习惯、知识背景、审美偏好，都和别人不一样。一个真正智能的 Agent，应该能"认识"你，并且根据你的特点来调整自己的行为。

Hermes 的**用户建模系统**会为你建立一个多维度的画像：

**基础画像**：你的角色（开发者/产品经理/投研人员等）、技术栈偏好、常用工具、工作时间规律等。

**交互偏好**：你喜欢简洁的回答还是详细的解释？你喜欢代码示例还是文字描述？你喜欢直接给答案还是引导你自己思考？

**知识图谱**：你熟悉哪些领域？在哪些领域你是新手？你经常提及的人物、公司、概念之间的关系是什么？

**任务模式**：你经常做什么类型的事情？你的任务有什么规律？（比如"每周一上午看周报"、"每天下午 3 点查数据"）

**反馈历史**：你对 Agent 的哪些回答满意？哪些不满意？你的修改习惯是什么？

基于这个画像，Hermes 会动态调整它的行为：
- 给开发者的回答会包含更多技术细节和代码示例
- 给产品经理的回答会更多关注用户场景和业务价值
- 如果你是个急性子，它会优先给出结论，细节放在后面
- 如果你是个谨慎的人，它会给出更多选项和权衡分析

> **Tips**
>
> 用户建模是完全本地化的，你的画像数据只存储在你自己的机器上。Hermes 承诺不会上传任何用户画像数据到第三方服务器。这也是选择本地 Agent 的重要理由之一。

### 3.3 和 OpenClaw 的定位差异

既然 Hermes 是从 OpenClaw 演进而来的，那它们之间到底是什么关系？你应该选哪个？

简单来说：**OpenClaw 是"探索者"，Hermes 是"定居者"。**

OpenClaw 在 2025 年横空出世，带着实验性和探索性的基因。它的目标是"证明 AI Agent 可以做什么"。它设计了很多酷炫的功能，但部分功能在稳定性和实用性上还有提升空间。它的架构相对灵活（也可以说是"松散"），社区贡献的技能质量参差不齐。

Hermes 则是站在 OpenClaw 的肩膀上，进行了一次彻底的架构升级。它的目标是"让 AI Agent 真正可用、好用、耐用"。它在以下方面做了重大改进：

| 维度 | OpenClaw | Hermes |
|------|----------|--------|
| 记忆系统 | 基于文件的简单存储 | 三层记忆系统，支持全文检索和语义检索 |
| 技能系统 | 静态技能，手动编写 | 自动创建 + 自我进化 |
| 学习机制 | 有限的反馈学习 | 完整的自学习闭环 |
| 用户建模 | 无 | 多维度用户画像 |
| 架构设计 | 模块化但耦合度较高 | 五层松耦合架构，清晰的分层边界 |
| 性能优化 | 基础 | 异步执行、并发处理、缓存机制 |
| 工具生态 | 约 30 个内置工具 | 47+ 内置工具 + MCP 生态 |

> **注意**
>
> 详细的对比分析将在第十三册「OpenClaw 对比与迁移」中展开。这里只是给你一个初步印象。如果你已经是 OpenClaw 用户，不需要担心——迁移到 Hermes 的过程比你想象的简单得多，而且大部分技能都可以兼容。

### 3.4 适合什么样的人

说了那么多，你可能还在想："Hermes 到底适不适合我？"

让我给你一个直截了当的答案。

**Hermes 特别适合以下人群：**

**第一类：每天需要跟 AI 协作的重度用户**

如果你每天要花 2 小时以上跟 AI 打交道——写代码、查资料、写文档、做分析——那么 Hermes 的自学习和个性化能力会给你带来巨大的效率提升。使用一个月后，你会发现它越来越"懂"你，很多任务只需要说几个字就能完成。

**第二类：对数据隐私有高要求的用户**

如果你处理的是商业机密、个人隐私、法律文件等敏感信息，Hermes 的本地化部署是刚需。你的数据永远不会离开你的机器，这在合规性和安全性上是无可替代的优势。

**第三类：希望打造个性化 AI 助手的极客**

如果你喜欢折腾技术，喜欢定制和扩展，Hermes 的技能系统和开源架构会给你很大的发挥空间。你可以写自己的技能、接入自己的工具、甚至修改核心代码。

**第四类：希望用 AI 创业的开发者**

Hermes 的灵活性和可扩展性，使它成为构建 AI 产品的优秀底座。无论是做 SaaS 服务、做企业解决方案，还是做垂直领域的 Agent 应用，Hermes 都能提供坚实的技术基础。

**第五类：从 OpenClaw 迁移的用户**

如果你对 OpenClaw 感到满意，但希望获得更好的记忆管理、更智能的技能系统和更稳定的性能，Hermes 是最好的选择。迁移成本低，收益明显。

**Hermes 可能不太适合：**

- **偶尔用一下 AI 的轻度用户**：如果你只是偶尔问几个问题，ChatGPT 或 Claude 的网页版可能更方便。
- **完全不想折腾的"纯小白"**：虽然本书会尽量做到保姆级，但 Hermes 毕竟需要安装和配置，比打开一个网页要复杂一些。不过，如果你愿意花 1-2 小时跟着本书操作，这个门槛完全可以跨越。
- **追求"开箱即用"极致体验的用户**：Hermes 的哲学是"给你控制权"，而不是"替你做好一切决定"。如果你喜欢"什么都不用管，厂商全部包办"的体验，闭源的云端服务可能更适合你。

> **Tips**
>
> 如果你还在犹豫，我的建议是：**先试试看**。按照本书第五册的安装指南，花 30 分钟把 Hermes 跑起来，然后实际体验一下。很多时候，"用起来"的感觉比"听起来"的判断要准确得多。而且，本地部署不花一分钱（除了 API 调用的费用，但如果你接本地模型，这个费用也是零），试试没有损失。

### 4.2 不同读者推荐的阅读路线

在 00-目录与导读 中，我设计了四条推荐阅读路线。这里再针对 01 册的读者做一个更详细的说明。

### 4.3 学习方法建议：边学边练、先跑通再深入

基于我带过几百个学员的经验，我总结了三个最有效的学习方法。

**方法一：边学边练，不要只看不练**

技术类知识有一个特点："看懂"和"会做"之间有一道鸿沟。你看懂了安装步骤，不代表你能一次安装成功。你看懂了配置说明，不代表你能配出最适合自己的方案。

所以，我强烈建议：每读完一册，至少动手实践一次。如果你读的是第五册（安装部署），那就真的去安装一遍。如果你读的是第六册（基础使用），那就真的去跟 Agent 对话几次。实践过程中遇到的问题，往往是最有价值的学习机会。

> **注意**
>
> 动手实践时，一定要做好记录。用一个文档或者笔记本，记录你执行的每一步命令、遇到的每一个错误、解决的每一个问题。这些记录不仅是你自己的"错题本"，将来也可以分享给其他遇到同样问题的人。

**方法二：先跑通，再深入**

很多技术学习者有一个共同的毛病：追求完美。他们希望在"完全理解"之后再动手，结果永远停留在"准备阶段"。

我的建议是：**先让系统跑起来，不管它是不是"最优配置""。**

用默认配置先把 Hermes 跑通，看到它能工作，建立信心。然后再逐步调整参数、优化配置、深入理解原理。"跑通"带来的成就感，会推动你继续学习下去。

**方法三：建立自己的知识网络**

不要把每一册当作孤立的知识点。Hermes 的 16 册内容是一张知识网络，册与册之间有很多关联。

比如，你学第五册（安装部署）时，会用到第四册（架构解析）中提到的某些概念。你学第八册（技能系统）时，会用到第七册（记忆系统）中讲到的持久化机制。你学第十四册（变现路径）时，会用到第二册（行业全景）中分析的市场趋势。

我建议你在学习过程中，画一张自己的"知识地图"——每学到一个新概念，就想想它跟之前学过的什么内容有关联。这种主动构建联系的学习方式，比被动阅读的效果要好得多。

> **Tips**
>
> 加入社区是学习的重要一环。Hermes 的中文社区虽然还在发展中，但已经有很多热心的用户在分享经验。遇到问题不要独自纠结，去社区里问一问，往往能得到更快的解答。第十五册会详细介绍社区资源的获取方式。

## 6. 术语表与约定

### 6.1 本书常用术语解释

为了让你在阅读后续章节时不被术语困扰，这里提前解释本书中会频繁出现的核心概念。

**Agent（智能体）**

能够感知环境、做出决策、执行行动并从中学习的 AI 系统。你可以把它理解为"有自主行动能力的 AI"。

**LLM（大语言模型）**

Large Language Model 的缩写。指那些参数量巨大、能理解和生成自然语言的 AI 模型，如 GPT-4、Claude、DeepSeek 等。你可以把它理解为 Agent 的"大脑"。

**Skill（技能）**

Agent 能执行的具体任务模块。比如"发送邮件"是一个技能，"搜索网页"是另一个技能。Skill 可以被创建、存储、调用和进化。类比一下：Skill 就像是助理掌握的"手艺"——有的助理会写代码，有的会写文案，有的会做表格。

**Memory（记忆）**

Agent 存储和检索信息的能力。Hermes 有三层记忆系统，让 Agent 能"记住"过去的事情。类比：Memory 就是助理的笔记本和大脑。

**MCP（Model Context Protocol）**

模型上下文协议。这是一个开放标准，用于统一 AI 模型与外部工具/服务之间的通信方式。类比：MCP 就像是 Agent 世界的 "USB-C" 接口——一个标准，连接万物。

**Gateway（网关）**

Hermes 中负责接入不同平台和协议的模块。比如微信 Gateway、飞书 Gateway、Telegram Gateway 等。类比：Gateway 就像是助理的"前台"——不同的人（平台）通过前台找到助理。

**Honcho**

Nous Research 团队开发的一个用户身份管理和会话隔离系统。在 Hermes 中用于区分不同用户、管理用户权限和隔离会话数据。类比：Honcho 就像是助理公司的"HR 系统"——管理客户信息、分配权限、确保保密性。

**RAG（检索增强生成）**

Retrieval-Augmented Generation 的缩写。一种让 AI 在生成回答之前先从知识库中检索相关信息的技术。类比：RAG 就像是助理在回答你之前，先去档案室查资料，确保回答准确。

**Prompt（提示词）**

你给 AI 的输入文本。好的 Prompt 能让 AI 给出更准确的回答。类比：Prompt 就是你给助理的"任务说明"。说明越清楚，助理执行得越好。

**Context Window（上下文窗口）**

LLM 一次能处理的最大文本长度。超出这个长度的信息会被截断或遗忘。类比：上下文窗口就是助理的"短期记忆容量"——能同时记住多少事情。

**Embedding（嵌入/向量）**

将文本转换成数字向量的技术。语义相似的文本，其向量在数学上也很接近。这是实现语义检索的基础。类比：Embedding 就像是给每句话生成一个"指纹"——意思相近的话，指纹也相似。

**FTS5**

SQLite 数据库的一个全文检索扩展。Hermes 用它来快速搜索持久记忆中的文本内容。类比：FTS5 就是档案室的"检索系统"——让你能快速找到需要的文件。

**Tool（工具）**

Agent 可以调用的外部功能。比如搜索工具、文件读写工具、邮件发送工具等。Hermes 内置了 47 个工具，还可以通过 MCP 接入更多。类比：Tool 就是助理能使用的"办公设备"——电话、电脑、打印机等等。

**Cron（定时任务）**

Linux/Unix 系统中用于定时执行任务的工具。在 Hermes 中用来设置自动化流程，比如"每天早上 8 点给我发日报"。类比：Cron 就是助理的"闹钟和日程表"。

**Sandbox（沙箱）**

一个隔离的执行环境。在沙箱中运行的代码不会影响主系统，即使代码有恶意行为或出现错误，也是安全的。类比：Sandbox 就是助理的"练习室"——在里面试错不会影响到真实业务。

**Docker**

一种容器化技术，可以把应用及其依赖打包成一个独立的容器，在任何环境中一致运行。类比：Docker 就像是"标准化集装箱"——不管里面装什么，都能在任何港口装卸。

**API（应用程序接口）**

不同软件之间交互的约定方式。比如 Hermes 可以通过 OpenAI 的 API 调用 GPT-4。类比：API 就是不同公司之间的"服务协议"——规定了怎么请求服务、怎么接收结果。

**Token**

LLM 处理文本时的最小单位。一个中文词通常对应 1-2 个 token，一个英文单词通常对应 1 个 token。API 调用的费用通常按 token 数量计算。类比：Token 就像是文字的"积木块"——模型一块一块地处理和理解文本。

**System Prompt（系统提示词）**

在每次对话开始时发给 LLM 的隐藏指令，用于设定 AI 的角色、行为规则和约束条件。用户通常看不到 System Prompt，但它深刻影响着 AI 的回答风格。类比：System Prompt 就是给助理的"员工手册"——规定了工作中应该遵守的准则。

**Workflow（工作流）**

一系列按特定顺序执行的步骤。在 Agent 中，Workflow 定义了完成某个任务的完整流程。类比：Workflow 就是助理的"标准作业程序（SOP）"——每一步该做什么，写得清清楚楚。

**Multi-Agent（多智能体）**

多个 Agent 协同工作的系统。不同的 Agent 扮演不同的角色，通过协作完成复杂任务。类比：Multi-Agent 就是一个"团队"——产品经理、设计师、工程师各司其职，共同完成项目。

**Fine-tuning（微调）**

在预训练好的大模型基础上，用特定领域的数据进一步训练，让模型更适合特定任务。Hermes 本身不涉及 fine-tuning，但了解这个概念有助于理解模型能力。类比：Fine-tuning 就是"岗前培训"——通用人才经过针对性培训后更适合特定岗位。

**Vector Database（向量数据库）**

专门用于存储和检索向量数据的数据库。在 RAG 系统中用于存储文本的 embedding，支持基于语义相似度的快速检索。Hermes 使用 SQLite 配合扩展来实现类似功能。类比：Vector Database 就是"按意思分类的档案柜"——你不需要知道文件标题，只要描述你想找什么，它就能找到相关的。

> **Tips**
>
> 这个术语表不需要一次性全部记住。建议你把它当作"词典"来用——读到某个术语不理解时，翻回来查一查。随着你阅读的深入，这些术语会变得越来越熟悉。
